使用hugo在github创建博客

2020年10月12日
| hugo踩坑

文章目录

本地安装hugo

参考： https://github.com/gohugoio/hugo

brew方式(mac)

# Homebrew (macOS)
brew install hugo

# 验证安装
hugo version

golang方式

需要本地有golang环境，至少go1.11版本，安装方式自行百度

  • 直接使用go命令安装
# 开启go mod
export GO111MODULE=on

# 下载安装
go get -v github.com/gohugoio/hugo

# 执行完成后可以$GOPATH/bin下有对应的可执行文件，如果不成功，可到源码根目录执行，go install

# 验证安装
hugo version
  • 使用git和go命令方式
# 下载源码
git clone https://github.com/gohugoio/hugo.git

# 切到根目录
cd hugo

# 安装
go install

# 验证安装
hugo version

创建博客

所有的命令，都在站点根目录下执行

生成新的站点

blog 是创建的站点对应的文件夹名称

# 生成站点
hugo new site blog

生成的目录：

  • archetypes: 默认的格式
  • content: 文档目录
  • data
  • layouts
  • resources: 资源目录
  • status: 静态资源目录
  • themes: 主题目录
  • config.toml: 配置目录 baseURL是站点的域名。title是站点的名称。theme是站点的主题。

主题配置

由于没有默认的主题，需要手动配置主题，可以去官方主题下载，也可以使用我的的主题

下载后，放到themes文件夹中，首次使用可以将主题文件夹下的exampleSite/config.toml复制到站点根目录，修改对应配置。

为了方便更新使用，我使用的git的submodule方式(官网介绍),已我使用的主题为例

# 初始化项目为git仓库
git init

# 创建submodule，并下载文件
git submodule add https://github.com/flysnow-org/maupassant-hugo themes/maupassant

# 复制配置文件，配置文件修改参考主题官网
cp themes/maupassant/exampleSite/config.toml .

评论配置

可以使用disques或者utteranc，disques需要注册账户，utteranc不需要的，对比之后选择使用utteranc用作评论

以我使用的主题为例

# disqusShortname = "leesirc"  ## 是否开启disqus评论，不要和utteranc同时开启

## 配置 utteranc评论,教程参考 
# https://utteranc.es/
# https://nusr.github.io/post/2019/2019-05-04-utterances-introduce/
[params.utteranc]
  enable = true
  repo = "<评论仓库地址>" ##换成自己得
  issueTerm = "pathname"
  theme = "github-light"

新建文章

建议使用命令方式创建，手动创建，可能会出现格式问题

hugo new post/first-blog.md

文件生成的位置在 content/post/first-blog.md，如果和我使用同一主题，必须放在content/post文件夹下。

启动本地服务

hugo server -D

启动后，访问localhost:1313即可看到效果，修改文件，会实时生效，方便预览。

github同步

需要用户github账号

  1. 在github上创建git仓库
  2. 在站点根目录创建**.gitignore**文件，可使用下面内容：
### Hugo ###
# 如果使用submodule关联public，就不需要忽略public
# /public/
/resources/_gen/
# 如果使用submodule关联主题，就不需要忽略themes
# /themes/

# Executable may be added to repository
hugo.exe
hugo.darwin
hugo.linux

# OSX
.DS_Store

关联站点项目仓库

git remote add origin <git仓库地址>

与github同步站点文件

  1. 生成博客文件
hugo -D

可以在public下看到生成的html文件 2. 创建git仓库，仓库名为 [username].github.io，配置仓库参考官网 3. 删除public文件夹，使用git的submodule方式关联仓库

git submodule add <仓库地址> public
  1. 重复第1步，github有延迟，大概一分钟访问 [username].github.io 即可看到。
